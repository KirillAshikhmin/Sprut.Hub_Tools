# 👁️ Скрыть устройство если выключено
Логический сценарий

> **Автоматическое скрытие выключенных устройств в интерфейсе**

Логический сценарий для Sprut.Hub, который автоматически скрывает устройства в интерфейсе, когда они выключены (или находятся в неактивном состоянии). Используется для создания комнаты-индикатора, куда добавляются виртуальные копии всех устройств, за которыми нужен контроль.

---

## 💡 Зачем это нужно?

**Проблема:** В интерфейсе отображается много устройств, но нужны только те, которые сейчас активны/включены. Лишние устройства засоряют интерфейс и мешают быстро найти нужное.

**Решение:** Сценарий автоматически скрывает устройства, когда они выключены или находятся в неактивном состоянии. Таким образом, в интерфейсе остаются только активные устройства, что упрощает навигацию и контроль.

---

## 📋 Пошаговая инструкция

### Шаг 1: Создание виртуальных копий устройств

Для каждого устройства, которое нужно контролировать, создайте виртуальную копию:
1. Откройте настройки устройства
2. Создайте виртуальную копию (если еще не создана)
3. Добавьте виртуальную копию в комнату-индикатор

### Шаг 2: Применение логического сценария

1. Скачайте файл `Скрыть устройство если выключено.json` из папки сценария
2. Импортируйте сценарий через интерфейс Sprut.Hub:
   - Сценарии → + → Импортировать шаблон → выберите скачанный файл
3. Откройте настройки виртуальной копии устройства
4. Перейдите в раздел **"Логика"**
5. Выберите сценарий **"Скрыть устройство если выключено"**
6. Активируйте сценарий переключателем "Активно"

### Шаг 3: Создание комнаты-индикатора

1. Создайте новую комнату (например, "Индикаторы" или "Активные устройства")
2. Добавьте в эту комнату все виртуальные копии устройств
3. Примените сценарий к каждому виртуальному устройству в комнате

## 🔧 Поддерживаемые типы устройств

Сценарий работает со следующими типами устройств:

- **Лампочки** (Lightbulb)
- **Выключатели** (Switch)
- **Телевизоры** (Television)
- **Окна** (Window)
- **Датчики контакта** (ContactSensor)
- **Двери** (Door)
- **Вентиляторы** (Fan, FanBasic)
- **Гаражные ворота** (GarageDoorOpener)
- **Клапаны** (Valve)
- **Краны** (Faucet)
- **Очистители воздуха** (AirPurifier)
- **Розетки** (Outlet)
- **Увлажнители/осушители** (HumidifierDehumidifier)
- **Оконные покрытия** (WindowCovering)

## 📊 Логика работы для разных характеристик

Сценарий определяет видимость устройства в зависимости от типа характеристики:

### HC.On (Включение/выключение)
- **Видимо:** когда устройство включено (`value = true`)
- **Скрыто:** когда устройство выключено (`value = false`)

### HC.Active (Активность)
- **Видимо:** когда устройство активно (`value = 1`)
- **Скрыто:** когда устройство неактивно (`value ≠ 1`)

### HC.CurrentPosition (Текущая позиция)
- **Видимо:** когда позиция = 100% (полностью открыто/поднято)
- **Скрыто:** когда позиция < 100%

### HC.ContactSensorState (Состояние датчика контакта)
- **Видимо:** когда контакт закрыт (`value = 1`)
- **Скрыто:** когда контакт открыт (`value ≠ 1`)

### HC.CurrentDoorState (Состояние двери)
- **Видимо:** когда дверь закрыта (`value = 0`)
- **Скрыто:** когда дверь открыта (`value ≠ 0`)

## 💼 Примеры использования

### Пример 1: Комната-индикатор включенных ламп

1. Создайте виртуальные копии всех ламп в доме
2. Добавьте их в комнату "Включенные лампы"
3. Примените сценарий к каждой виртуальной копии
4. В комнате будут отображаться только включенные лампы

### Пример 2: Индикатор открытых окон

1. Создайте виртуальные копии всех окон
2. Добавьте их в комнату "Открытые окна"
3. Примените сценарий к каждой виртуальной копии
4. В комнате будут отображаться только открытые окна (позиция = 100%)

### Пример 3: Контроль активных устройств

1. Создайте виртуальные копии всех устройств, которые нужно контролировать
2. Добавьте их в комнату "Активные устройства"
3. Примените сценарий к каждой виртуальной копии
4. В комнате будут отображаться только активные устройства

---

## ⚠️ Важные замечания

1. **Виртуальные копии:** Сценарий работает только с виртуальными копиями устройств, не с оригинальными устройствами. Это сделано для безопасности, чтобы не скрывать реальные устройства.

2. **Автоматическое обновление:** Сценарий автоматически обновляет видимость при каждом изменении состояния устройства благодаря параметру `onStart: true`.

3. **Типы устройств:** Сценарий поддерживает множество типов устройств, но логика работы зависит от типа характеристики. Убедитесь, что устройство имеет поддерживаемую характеристику.

4. **Комната-индикатор:** Рекомендуется создать отдельную комнату для индикаторов, чтобы не путать виртуальные копии с реальными устройствами.

---

## 🔄 Как это работает

1. При изменении состояния устройства (включение/выключение, открытие/закрытие и т.д.) срабатывает функция `trigger`
2. Сценарий определяет тип характеристики устройства
3. В зависимости от типа характеристики и текущего значения определяется видимость устройства
4. Устройство скрывается или показывается в интерфейсе автоматически

---

## 📝 Примечания

- Сценарий работает только с виртуальными копиями устройств
- Видимость обновляется автоматически при изменении состояния устройства
- Сценарий не требует настройки параметров - работает "из коробки"
- Поддерживает множество типов устройств и характеристик
